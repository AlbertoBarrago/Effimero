Getting started

From nothing to your first numbers in about five minutes.

1. Start the server

Effimero is one Docker image plus Redis. Clone, copy the sample env, and bring the stack up:

git clone https://github.com/albertobarrago/effimero && cd effimero
cp .env.sample .env
docker compose up -d
curl http://localhost:3000/health   # {"status":"ok","redis":true}

2. Register the site

/collect only accepts hits for registered sites; an unknown siteId is silently ignored, so nobody can inject stats for a site you never set up. Register it with the admin key (see step 4 for where that key comes from). You can restrict which origins may send hits with allowedOrigins — leave it empty to accept any:

curl -X POST https://your-host/admin/sites \
     -H "Authorization: Bearer <STATS_API_KEY>" \
     -H "Content-Type: application/json" \
     -d '{"siteId":"my-site","allowedOrigins":["https://my-site.com"]}'

The response includes a per-site read token, shown only once — store it now:

{ "siteId": "my-site", "allowedOrigins": ["https://my-site.com"],
  "active": true, "createdAt": "…", "readToken": "…store-this-now…" }

List registered sites with GET /admin/sites, change the origins with PATCH /admin/sites/my-site, rotate a lost token with POST /admin/sites/my-site/token, and remove a site with DELETE /admin/sites/my-site — all with the admin key.

3. Add the snippet

Place this tag in your page <head> or before </body>:

<script src="https://your-host/effimero.js" data-site="my-site" defer></script>
AttributeRequiredDescription
data-siteyesSite identifier. Letters, digits, . _ -, max 64 chars. Must match a registered site.
data-endpointnoCollect URL. Defaults to /collect on the snippet's host.

The snippet weighs about 700 bytes, tracks SPA navigations (pushState and popstate), and disables itself when the browser sends Do Not Track or Global Privacy Control.

4. Unlock the dashboard

Read endpoints accept two credentials, both passed as Authorization: Bearer <value>:

  • the STATS_API_KEY (admin) — reads every site and manages the registry;
  • a per-site read token (from step 2) — reads only its own site. It is rejected with 403 for any other site, so it is safe to hand to a client.

On first boot Effimero generates the admin key and logs it:

docker compose logs effimero | grep generated
# STATS_API_KEY not set, generated one for this run: <key>

Open the dashboard at http://localhost:3000/ or https://your-host/, paste either credential when prompted, and you are in. The generated admin key changes on every restart; set a stable one in .env:

STATS_API_KEY=change-me

5. Watch the data arrive

Enter your site id and you will see live visitors, daily uniques and pageviews, top pages and referrers, browsers, OS, devices, languages, countries, and pageviews by hour. Prefer the API? Query it with either credential:

curl https://your-host/stats/my-site?range=30 \
  -H "Authorization: Bearer <token-or-admin-key>"
